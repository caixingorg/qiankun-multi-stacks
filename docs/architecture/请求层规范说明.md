# 请求层规范说明

## 一、当前定位

请求层规范的目标不是让每个子应用自己决定认证与环境注入，而是由 `main` 提供统一请求入口。

当前由 `main/src/services/request-client.js` 负责定义 Host 请求约定。

## 二、当前规则

1. 请求入口由 `main` 提供
2. 子应用如果要共享主应用注入的请求能力，统一通过 `sharedKernel.request`
3. 子应用自己的 `services/api.js` 允许保留本域接口描述，但要对齐主应用注入的认证和环境语义
4. `window.__MAIN_AUTH_TOKEN__` 只允许由主应用登录流程或 BFF 注入，子应用不得自己写全局 token

## 三、当前字段约定

当前 Host request 行为至少包含：

1. `fetch` 发起真实请求
2. `Authorization` 头懒读 `window.__MAIN_AUTH_TOKEN__`
3. `x-user-id`
4. `x-env-name`
5. `x-auth-owner`
6. 请求超时控制
7. 统一非 2xx 错误语义
8. `401` 统一使用 `AUTH_EXPIRED` 前缀

其中当前默认请求头包含：

1. `x-user-id`
2. `x-env-name`
3. `x-auth-owner`

## 四、后续要求

后续如果接真实接口，请继续保持：

1. 认证头由主应用统一注入
2. 环境头由主应用统一注入
3. 子应用不直接碰全局 token
4. 子应用域内 `api.js` 以“资源名 + 路径 + 调用函数”组织
5. Host request 的超时、401 语义、错误格式不能被子应用各自改写
