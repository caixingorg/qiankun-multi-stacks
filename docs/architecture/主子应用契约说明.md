# 主子应用契约说明

## 一、目标

这份文档用于明确主应用 `main` 与子应用 `subapps/*` 的边界，避免后续继续出现职责漂移、直接耦合、事件乱飞的问题。

## 二、菜单契约

1. 一级菜单归 `main` 所有
2. 子应用只消费主应用传入的菜单上下文
3. 子应用不得接管主应用一级导航

## 三、路由契约

1. `main` 负责子应用激活规则
2. 子应用负责自身内部路由
3. 子应用不得直接改写主应用激活规则

## 四、权限契约

1. 全局权限上下文由 `main` 持有
2. 子应用消费权限信息
3. 子应用可以细化页面级权限，但不能改变全局权限模型

## 五、上下文契约

当前主应用传给子应用的最小上下文包括：

- `userContext`
- `envContext`
- `permissionContext`
- `navigation`
- `sharedKernel`
- `dependencyPolicy`
- `contractVersion`

## 六、通信契约

当前允许的主子应用事件包括：

- `subapp:notify`
- `subapp:runtime-error`
- `subapp:navigation-request`
- `host:broadcast`
- `host:navigation`
- `host:navigation:guard`

新增事件必须先补到共享常量层，再允许进入代码。

## 七、错误契约

1. 子应用错误必须可被主应用识别
2. 主应用错误不能直接污染子应用业务状态
3. 错误上报字段应至少包含应用名、环境、消息摘要
