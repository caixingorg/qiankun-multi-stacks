# 错误监控与日志方案说明

## 一、当前定位

当前模板已经把日志和错误的入口集中到了 `main`：

- `main/src/services/logger.js`
- `main/src/services/error-reporter.js`

## 二、规则

1. 主应用负责跨应用运行期错误面
2. 子应用运行错误必须能被主应用识别
3. 日志最小字段必须带 `scope` 和 `appName`
4. 后续如果接监控平台，也先从 `main` 接入
5. startup fatal error 与 subapp runtime error 必须区分

## 三、当前最小日志字段

1. `scope`
2. `appName`
3. `eventName`
4. `ts`
5. `payload`

## 四、当前最小错误字段

1. `scope`
2. `stage`
3. `channel`
4. `activeSubApp`
5. `envName`
6. `severity`
7. `rawMessage`

## 五、后续扩展

3. 子应用加载失败统计
4. rollback 自动切换日志
