# 菜单路由权限联动说明

## 一、当前定位

菜单、路由、权限三者当前由 `main` 统一协调。

相关代码：

- 菜单配置：`main/src/navigation/menu-config.js`
- 菜单过滤：`main/src/navigation/menu-permission.js`
- 主应用路由解析：`main/src/routes/`

## 二、联动规则

1. 菜单项定义主入口路径
2. 菜单项定义对应权限码
3. 路由访问判断复用同一套权限映射
4. 子应用内部页面切换不反向控制主应用一级菜单

## 三、当前最小联动模型

当前主应用里：

1. `mainMenuConfig` 同时定义 `path` 与 `permission`
2. `routePermissionMap` 从菜单配置直接生成
3. `canAccessRoute` 用于判断主路由是否允许访问

## 四、后续扩展方向

后续如果接真实权限系统，仍然建议维持：

1. 一级菜单归 `main`
2. 激活路由归 `main`
3. 子应用只细化二级页面和操作级权限
