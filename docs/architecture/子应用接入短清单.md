# 子应用接入短清单

这份清单只给接入团队直接使用，只回答 5 件事：

1. 主应用传什么
2. 子应用怎么引用、怎么用
3. 生命周期怎么接
4. cleanup 要做什么
5. 常见错误先查哪里

## 1. 主应用传什么

主应用通过 qiankun `props` 注入的最小能力固定为：

1. `actions`
2. `bus`
3. `sharedKernel`
4. `userContext`
5. `envContext`
6. `permissionContext`
7. `navigation`
8. `dependencyPolicy`
9. `contractVersion`

接入团队只消费这些字段，不直接依赖主应用源码路径。

## 2. 子应用怎么引用、怎么用

子应用只允许引用自己仓内的 bridge 文件，例如：

1. `src/bridge/micro-app.js`
2. `src/bridge/standalone.js`
3. `src/bridge/host-api.js`
4. `src/bridge/get-host-context.js`

禁止在子应用里直接 import：

1. `main/src/shared/*`
2. `main/src/runtime/*`
3. 任何 `../../../../main/src/*`

推荐做法：

1. `host-api.js` 负责读取 `props`、校验主应用注入字段、解析 runtime vendor、承接 qiankun lifecycle 辅助。
2. `micro-app.js` 只负责 `buildViewModel(props)`、调用 `renderApp()`、导出 `bootstrap / mount / unmount`。
3. `standalone.js` 只负责独立运行兜底视图和回主应用跳转。

## 3. 生命周期怎么接

最小接法：

1. `bootstrap` 只保留启动日志或必要预热。
2. `mount(props)` 先执行 `assertMainContract(props)`。
3. `mount(props)` 里先注册全局状态和 host 广播，再执行子应用自己的渲染。
4. `unmount()` 先执行子应用卸载，再统一 flush cleanup。

如果是 Vite 子应用：

1. 在 `micro-app.js` 里用 `renderWithQiankun(...)` 包住本地 lifecycle。

如果是 Webpack 子应用：

1. 直接导出本地 lifecycle 返回的 `bootstrap / mount / unmount`。

## 4. cleanup 要做什么

所有 mount 阶段新增的副作用，都要进入 cleanup：

1. `addEventListener`
2. `setInterval`
3. `setTimeout`
4. `requestAnimationFrame`
5. `actions.onGlobalStateChange`
6. `bus.on(...)`

原则只有一条：

1. mount 里创建的浏览器副作用，unmount 时必须能一次性清掉。

## 5. 常见错误查哪里

### 拿不到主应用上下文

按这个顺序查：

1. `main/src/runtime/register-subapps.js`
2. 子应用自己的 `src/bridge/host-api.js`
3. 子应用自己的 `src/bridge/micro-app.js`

### contract 校验失败

先查：

1. `contractVersion`
2. `userContext / envContext / permissionContext`
3. `navigation / sharedKernel / dependencyPolicy`

### 跨应用跳转不对

先查：

1. 主应用 `main/src/navigation/`
2. 子应用 `buildViewModel(props)` 里 `navigation.navigate(...)` 的入参

### runtime vendor 不可用

先查：

1. `main/public/runtime/runtime-vendor.global.js`
2. 主应用 `main/src/runtime/runtime-vendor/`
3. 子应用自己的 `resolveRuntimeVendor()` 调用

### 子应用加载了但退出不干净

先查：

1. 子应用 `src/bridge/host-api.js` 里的 cleanup 收口
2. 子应用 `micro-app.js` 的 `onMount` 是否注册了未清理副作用
3. 子应用 `onUnmount` 是否只做视图卸载，遗漏了浏览器资源释放
