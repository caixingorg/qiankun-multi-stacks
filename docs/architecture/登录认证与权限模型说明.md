# 登录认证与权限模型说明

## 一、当前定位

这一版模板里：

1. 登录态由 `main` 持有
2. 权限模型由 `main` 持有
3. token 注入由 `main` 的请求层统一负责
4. 子应用只消费认证结果和权限结果

## 二、当前代码落点

- 认证模型常量：`shared/constants/auth-model.js`
- 用户上下文：`main/src/context/user-context.js`
- 权限上下文：`main/src/context/permission-context.js`
- 认证上下文：`main/src/context/auth-context.js`
- 主上下文聚合：`main/src/context/index.js`

## 三、规则

1. 子应用不得自行定义全局登录态来源
2. 子应用不得自行改写全局权限集合
3. 子应用只允许在页面内做二次细化，例如按钮级是否展示
4. 子应用自己的请求不得绕开主应用请求层约定去伪造认证来源

## 四、当前最小模型

当前最小认证模型包含：

1. `loginState`
2. `loginOwner`
3. `permissionOwner`
4. `tokenInjectionOwner`
5. `sessionId`
6. `envName`

## 五、后续扩展边界

后续如果要接真实登录系统，应继续由 `main` 统一接入：

1. SSO
2. token refresh
3. 菜单权限裁剪
4. 主应用级路由守卫

子应用仍然不直接接登录中心。
