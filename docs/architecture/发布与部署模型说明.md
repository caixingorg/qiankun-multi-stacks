# 发布与部署模型说明

## 一、环境划分

当前定义四个环境：

- `local`
- `test`
- `staging`
- `production`

## 二、入口来源

1. 主应用入口由 `main` 提供
2. 子应用入口默认注册表由 `main/src/runtime/app-registry.js` 维护
3. 环境相关 host 配置由 `main/public/config/env/*.json` 维护
4. 子应用 manifest 由 `main/public/config/apps/*.json` 维护
5. 非本地环境优先依赖运行时 manifest，不再允许自动回退到 localhost 默认注册表

## 三、构建入口

根目录统一入口：

- `npm run build:main`
- `npm run build:subapp:legacy`
- `npm run build:subapp:wms`
- `npm run build:subapp:ops`
- `npm run build:subapp:vite-vue`
- `npm run build:subapp:vite-react`

## 四、回滚模型

当前回滚能力仍然是应用级示例能力：

- `legacy`
- `ops`
- `viteReact`

仍为 dedicated rollback 参考路径。

以下应用仍是 shared-entry：

- `wms`
- `viteVue`

## 五、版本兼容

当前约束：

1. `main` 的共享上下文字段变更必须同步更新契约文档
2. 主应用给子应用传递的新字段必须进入共享常量或共享上下文说明
3. 子应用不得私自依赖主应用内部私有字段
4. 生产环境缺失 manifest 或 auth 注入时，主应用应 fail-closed，而不是回退到开发默认值
